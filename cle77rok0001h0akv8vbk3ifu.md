The method defines **network protocol**(a certain set of rules). Every method informs how the *submission* of a particular type of data can be handled.

HTML only handles two types of methods

1) GET-Whenever we submit the data, data gets <mark>passed to the URL</mark> and the server reads the data from that URL. The disadvantage of this method is data get saved in history and someone can misuse that data.

2) POST-Whenever we submit data using this method data get passed to a component known as the <mark> request body </mark> and the server read the data from this component body only its biggest advantage is data doesn't get saved in the component body.
